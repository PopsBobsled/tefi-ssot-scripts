# TEFI Client Lifecycle Model — Where Principles Apply

**Version:** 1.0 | **Last updated:** 2026-03 | **Owner:** Tate — TEFI

---

## PURPOSE

This document maps where each Communication Principle is applied across the client journey — from initial outreach through to post-engagement support.

It clarifies which emails, reports, and interactions matter most and what principles govern each stage.

---

## THE CLIENT JOURNEY — TIER 1 PATHWAY

```
STAGE 1: OUTREACH
└─ You reach out to prospect
   └─ Principle: Acknowledgement-First + Invitation-Based

STAGE 2: CV RECEPTION
└─ Client sends CV in response
   └─ Principle: Acknowledgement-First (receipt confirmation)
   └─ Email: Draft 1 — "Your CV — received and in process"

STAGE 3: INFORMATION GATHERING
└─ You request additional details via Tier 1 Action Prompt
   └─ Principle: Benefit-Before-Burden + Specificity + Question Taxonomy
   └─ Email: Draft 3 — "Your Tier 1 Career Migration Report — a few details..."
   └─ Client replies with details

STAGE 4: SNAPSHOT & ASSESSMENT
└─ You deliver Career Migration Snapshot
   └─ Principle: Source-of-Truth + Confidence Without Overstatement
   └─ Email: Draft 2 — "Your Career Migration Snapshot — New Zealand and Australia"
   └─ Document: Full Tier 1 report (6 sections, income estimates, trajectory)

STAGE 5: ENGAGEMENT INVITATION
└─ Offer optional conversation via Calendly
   └─ Principle: Invitation-Based Engagement
   └─ Channel: Email, Calendly link, optional reply

STAGE 6: UPGRADE DECISION (Tier 2+)
└─ Client decides to proceed or not
   └─ If proceed: Tier 2 CV upgrade begins
   └─ If not: Maintain relationship, periodic updates

STAGE 7: CONTINUOUS LEARNING
└─ Signal tags on improved emails
   └─ Principle: Monthly Signal Learning
   └─ Update: Communication Principles + Signal Registry
```

## THE THREE DRAFTS — PRINCIPLE MAPPING

### Draft 1 — Acknowledgement Email
**When:** Immediately after receiving client CV
**Principles applied:**
1. Acknowledgement-First Sequence (open with something specific from CV)
2. Tone — Warm, Direct, Coach-like
3. Invitation-Based Engagement (Calendly link, optional)
4. Benefit-Before-Burden (explains what's coming)

**Success criteria:**
- Client feels seen and understood
- Expectations are clear
- Next step is obvious but not pushy

---

### Draft 2 — Career Migration Snapshot Notification
**When:** When Snapshot PDF is ready to send
**Principles applied:**
1. Acknowledgement-First Sequence (open with acknowledgement of their profile)
2. Confidence Without Overstatement (describe what's in the report honestly)
3. Tone — Warm, Direct, Coach-like
4. Invitation-Based Engagement (offer to discuss)

**Success criteria:**
- Client understands what they're receiving
- Report feels personalised (not generic)
- Opportunity to deepen relationship is clear

---

### Draft 3 — Tier 1 Questions (Action Prompt)
**When:** Before building the Snapshot, request additional info
**Principles applied:**
1. Specificity-Over-Generality (every Q traces to CV)
2. Question Taxonomy — High vs Medium Value
3. Benefit-Before-Burden Framing (explain why each question matters)
4. Tone — Warm, Direct, Coach-like
5. Source-of-Truth Discipline (no invented gaps)

**Success criteria:**
- Every question is relevant and specific
- Client understands why the detail matters
- Response rate is high (clients see value in answering)

## EXTENDED JOURNEY — TIER 2+

If client moves to Tier 2 CV upgrade:

### New principles emerge:
- **Revision transparency** — Show before/after, explain each change
- **Quality assurance** — Confirm CV meets standards before delivery
- **Escalation clarity** — Explain what Tier 3/4 adds, without pressure

*(These principles will be captured as new signals emerge from Tier 2 work)*

**What Tate does:**

- Approve or refine the change

**What Claude does:**
- Log all signals with dates
- Provide monthly summary
- Flag any contradictions or exceptions

## GOVERNANCE RULES

**When a principle can be updated:**
- Signal-based learning only (tagged emails)
- With monthly review approval from Tate
- Changes documented in Signal Registry

**When a new principle is needed:**
- When a signal reveals something not covered by existing principles
- Escalate to Tate for discussion
- Add to Communication_Principles.md + Exception_Rules.md

**When a principle is wrong:**
- Document the contradiction
- Create an exception rule (see 04_Exception_Rules.md)
- Or revise the principle if the learning is broad enough

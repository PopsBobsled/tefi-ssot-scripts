# TEFI Exception Rules — When Principles Bend

**Version:** 1.0 | **Last updated:** 2026-03 | **Owner:** Tate — TEFI

---

## PURPOSE

This document captures cases where a Communication Principle doesn't apply, applies differently, or is deliberately overridden.

Exception rules prevent principles from becoming dogma. They're discovered through signal learning and approved by Tate.

---

## STRUCTURE OF AN EXCEPTION

Each exception has:
- **Principle it modifies** — Which principle from 01_Communication_Principles.md
- **Condition** — When this exception applies
- **Modified approach** — How the principle changes
- **Why** — The business/client logic
- **First signal** — Date and context where this was discovered
- **Approval** — Tate's sign-off

## EXAMPLE (FOR REFERENCE — NOT YET IN USE)

### Exception: Tone shift for highly technical profiles

**Principle affected:** Principle 4 — Tone (Warm, Direct, Coach-like)

**Condition:** Client is a senior technical architect or CTO with deep domain expertise who has expressed preference for direct, minimal-fluff communication.

**Modified approach:** Shift toward more technical register and shorter sentences. Reduce coach-like warmth slightly in favour of precision. Still warm, but more direct and less conversational.

**Why:** Over-personalisation can feel patronising to very senior technical profiles. They value efficiency and precision. Matching their communication style builds trust faster.

**First signal:** [Example: 2026-04-15, Client XYZ, preferred shorter email with technical terms vs. conversational framing]

**Approved by Tate:** [Pending]

**Notes:** Does not apply to all senior profiles — only those who signal explicit preference. Warm remains foundational; this is a relative shift only.

## GOVERNANCE

**Who can propose an exception:**
- Tate (when reviewing signals)
- Claude (when extracting signals that seem to contradict principles)

**How exceptions are approved:**
- Logged to Signal Registry
- Discussed in monthly review
- Added to this file only after Tate approval

**When exceptions sunset:**
- If the condition no longer applies (client type goes away, market shifts)
- If the exception reveals the principle itself was wrong (in which case, revise the principle instead of maintaining the exception)
